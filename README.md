# HeightMatch

This repository provides the training, testing, and synthetic height map generation pipeline for **HeightMatch**.  
Below, we use the **INRIA_Buildings** dataset as an example to describe the complete workflow:

1. Generate synthetic height maps for the dataset;
2. Train the model;
3. Test the trained model.

---

## 1. Overview

For the **INRIA_Buildings** dataset, the recommended workflow is:

- First generate synthetic height maps from RGB images;
- Then run the training script;
- Finally run the testing script.

Because the INRIA dataset is usually organized by **city**, and the height generation script only accepts one `imgdir` and one `outdir` each time, you need to run the generation script **multiple times** if the dataset contains multiple city folders.

## 3. Generate Synthetic Height Maps

Synthetic height maps are generated by:

```bash
generate_synthetic_height.py
```

This script reads RGB images from a given input folder and writes the generated height maps to a given output folder.

### 3.1 Example for one city

For example, for the `Austin` subset:

```bash
python3 generate_synthetic_height.py     --imgdir /home/Datasets/RSSeg/INRIA/Austin/image     --outdir /home/Datasets/RSSeg/INRIA/Austin/height
```

### 3.2 Run the script for all INRIA cities

If your INRIA dataset contains multiple city folders, run the script once for each city:

```bash
python3 generate_synthetic_height.py     --imgdir /home/Datasets/RSSeg/INRIA/Austin/image     --outdir /home/Datasets/RSSeg/INRIA/Austin/height

python3 generate_synthetic_height.py     --imgdir /home/Datasets/RSSeg/INRIA/Chicago/image     --outdir /home/Datasets/RSSeg/INRIA/Chicago/height

python3 generate_synthetic_height.py     --imgdir /home/Datasets/RSSeg/INRIA/Kitsap/image     --outdir /home/Datasets/RSSeg/INRIA/Kitsap/height

python3 generate_synthetic_height.py     --imgdir /home/Datasets/RSSeg/INRIA/Tyrol-W/image     --outdir /home/Datasets/RSSeg/INRIA/Tyrol-W/height

python3 generate_synthetic_height.py     --imgdir /home/Datasets/RSSeg/INRIA/Vienna/image     --outdir /home/Datasets/RSSeg/INRIA/Vienna/height
```

### 3.3 Output location

The generated height maps are saved to the folder specified by `--outdir`.

For example:

```text
Input image : /home/Datasets/RSSeg/INRIA/Austin/image/austin1.tif
Output map  : /home/Datasets/RSSeg/INRIA/Austin/height/austin1.png
```

The script preserves the relative folder structure under the input directory.

## 4. Prepare Split Files

Before training, you need to prepare the split files under the `splits` directory.

A typical structure is:

```text
splits/inria_buildings/
├── train.txt
├── val.txt
├── test.txt
└── 0.5/
    ├── labeled.txt
    └── unlabeled.txt
```

Where:

- `train.txt` contains all training samples;
- `val.txt` contains validation samples;
- `test.txt` contains test samples;
- `labeled.txt` contains labeled training samples for a given split ratio;
- `unlabeled.txt` contains unlabeled training samples for the same split ratio.

Each line should contain three relative paths:

```text
relative/path/to/image relative/path/to/label relative/path/to/height
```

For example:

```text
Austin/image/austin1.tif Austin/label/austin1.tif Austin/height/austin1.png
```

Make sure the third column points to the generated synthetic height maps.

## 5. Training

Training is launched with:

```bash
train.sh
```

In the script, you can set:

- `dataset`
- `split`
- `method`
- `model`

For INRIA_Buildings, an example setting is:

```bash
dataset='inria_buildings'
split='0.5'
method='heightmatch'
model='dinov2_small'
```

These variables define the following paths automatically:

```bash
config=configs/${dataset}.yaml
save_path=saved/$dataset/$split/$method/$model
labeled_id_path=splits/$dataset/$split/labeled.txt
unlabeled_id_path=splits/$dataset/$split/unlabeled.txt
val_id_path=splits/$dataset/val.txt
```

### 5.1 Start training

If `train.sh` has already been configured, you can start training with:

```bash
bash train.sh 4
```

Here, `4` means using 4 GPUs.

If you only want to use 1 GPU:

```bash
bash train.sh 1
```

### 5.2 Training output

The trained checkpoints will be saved to:

```text
saved/inria_buildings/0.5/heightmatch/dinov2_small/
```

Typically, this directory will contain:

- `latest.pth`
- `best.pth`

## 6. Testing

Testing is launched with:

```bash
test.sh
```

Before testing, make sure that the settings in `test.sh` match those used in training, for example:

```bash
dataset='inria_buildings'
split='0.5'
method='heightmatch'
model='dinov2_small'
```

### 6.1 Run testing

```bash
bash test.sh
```

### 6.2 Test behavior

The testing script automatically uses:

- the config file:
  ```text
  configs/inria_buildings.yaml
  ```
- the test split file:
  ```text
  splits/inria_buildings/test.txt
  ```
- the trained model directory:
  ```text
  saved/inria_buildings/0.5/heightmatch/dinov2_small
  ```

By default, the script loads:

```text
saved/inria_buildings/0.5/heightmatch/dinov2_small/best.pth
```

### 6.3 Test output

The test results are written to:

```text
test_results.xlsx
```

---

## 8. Notes

1. The synthetic height generation script processes only one `imgdir` and one `outdir` at a time.  
   Therefore, for datasets organized by multiple cities, you must run it multiple times.

2. Before training, make sure that all split files are correctly prepared and that the third column in each line points to the generated height map.

3. The testing script loads `best.pth` by default.  
   Please make sure this checkpoint exists before testing.

4. If you change `dataset`, `split`, `method`, or `model`, keep the training and testing scripts consistent.

5. If you use a different dataset structure, you only need to make sure that:
   - RGB images are available;
   - labels are available;
   - synthetic height maps are generated and stored in the paths listed in the split files.

---

```text
Generate synthetic height maps

        ↓

        ↓
```
